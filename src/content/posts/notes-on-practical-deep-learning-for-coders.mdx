<p><em>31 October 2023</em></p><p class="msg msg--info">Course: <a href="https://course.fast.ai/">https://course.fast.ai/</a></p><p class="msg msg--info">Book: <a href="https://github.com/fastai/fastbook">https://github.com/fastai/fastbook</a></p><h2>Intro</h2><p>Deep Learning is a computer technique to extract and transform data using multiple neural network layers.</p><p>Top used use cases:</p><ol><li>NLP: answer questions; speech recognition; summarize; classify; find; search</li><li>Computer Vision: satellite image interpretation, face recognition, caption</li><li>Medicine: finding anomalies; counting features in slides; measuring; diagnosing</li><li>Biology: classifying genomic tasks; analyzing cell interactions</li><li>Image Generation</li><li>Recommend Systems: search engines, home page layout, product recommendation</li><li>Playing Games</li><li>Robotics</li><li>Financial, Logistics, etc...</li></ol><p>In 1943, Warren McCulloch and Walter Pitts developed a mathematical model of artificial neurons. It's declared in "<em>A logical calculus of the ideas immanent in Neurous Activity</em>."</p><p>The neuron could be represented as addition and thresholding.</p><figure class="align-center"><img loading="lazy" src="https://www.researchgate.net/publication/258372123/figure/fig1/AS:374245319299072@1466238445241/Mathematical-model-of-artificial-neuron.png" alt="Mathematical model of artificial neuron | Download Scientific Diagram" width="239" height="166" data-is-external-image="true" /></figure><p>Frank Rosenblatt gave artificial neurons the ability to learn. He even created the first device on this principle: <strong>Mark I Perceptron</strong>.</p><blockquote><p>We are now about to witness the birth of such a machine - capable of perceiving, recognizing, and identifying the surroundings without any human training or context.</p><p class="align-right">"The Design of an Intelligent Automata"</p><p class="align-right">Frank Rosenblatt</p></blockquote><p>Marvin Minsky and Seymour Papert wrote a book called "<em>Perceptrons</em>." They showed that the device needs multiple layers to learn simple mathematical functions.</p><p>Perhaps the most pivotal work in neural networks was the "<em>Parallel Distributed Processes</em>" by David Rumelhart, James McCallen, and the PDP Research Group, released in 1986.</p><p>PDP approach for the programming framework defined requires:</p><ol><li>process unit sets</li><li>state of activation</li><li>output function for each unit</li><li>connectivity pattern among units</li><li>propagation rule for propagating through the network of connectivities</li><li>activation rule for combining inputs impinging on a unit with the current state of that unit to produce output for the unit</li><li>learning rule whereby patterns of connectivity are modified by experience</li><li>an environment where PDP must operate</li></ol><p>Modern Neural Networks use an approach similar to PDP.&nbsp;</p><p class="msg msg--info msg--highlight">The book says that the theoretical knowledge was misunderstood, and the issues held back the development until 2010. But also says that despite having two layers, it was too big and too slow to be helpful. Recent advancements in the hardware allowed training neural networks with more layers.</p><p class="msg msg--info">For learning, it is best to use Jupyter Notebooks. The Jupyter Notebooks are software that allows you to include assets and code together.</p><p>The deep learning uses the neural networks. It's part of machine learning. The deep learning model process and its training apply to the general concepts of machine learning.</p><p>Machine learning is another form of programming where instead of the algorithm, you write the desired output, and the software finds the path.</p><p>The term "machine learning" was coined by Arthur Samuel from IBM in 1949. In his 1962 classic essay "<em>Artificial Intelligence: A Frontier of Automation,</em>" he wrote machine technique.</p><p>Rather than describing each step to solve the task, show an example of the problem to solve. Then, let the program figure out the algorithm.</p><hr /><p>There is an actual performance of weight assignment. Then, there is an automatic means of testing performanceâ€”a mechanism for improving performance by changing weight assignments.</p><p>The weights are variables. The weight assignments are a choice of those variables. The program inputs are variables.&nbsp;</p><p>In modern terminology, machine learning programs are called models.</p><p>The weight assignment is the second input describing the actual input. In modern terminology, weight assignments are called model parameters.</p><p>Automatic testing means you let two models work against each other. Actual performance is the desired result. After each iteration, you adjust the model to test the outcome.</p><p>If you automate the iteration to adjust the model parameters, the program could learn from experience.</p><p>This automatic iteration is called model training.</p><p>Once the weights were adjusted, they became part of the model.&nbsp;</p><p>The neural networks are mathematical functions. The mathematical proof called the Universal Approximation Theorem shows this function can solve any problem in theory.</p><p>The general approach to adjusting weights is made via Stochastic Gradient Descent (SGD), a mathematical method.</p><p>The model includes weight, a function producing results, and performance updating weights. The functional part is called architecture.</p><p>Prediction is the result. The loss is the measurement of the performance. It's measured against the labels.</p><p>Transformers are code that is applied during the training. There are item and batch transformers. The inputs are not used to train. Some part of the set is kept separated to test that training.</p><p>The training input is called a training set, while testing data is used as a validation set. Over time, as the validation set is set, the model starts memorizing the labels. Thus, it doesn't find the generalizing patterns. This is called overfitting.</p><p>Overfitting is the single, most important issue in the model training.</p><p>Resnet is one of the architectures. It could have 18, 34, 50, 101 and 150 layers. The more layers mean training and with less data will be prone to overfitting.</p><p>The model that has trained weights is called pre-trained model. It's better to build on the pre-trained models by changing some layers with yours. Using pre-trained model for a task different than what it was originally trained is called transfer learning.</p><p>Fine tuning is a transfer learning technique where parameters of pretrained models update by additional epoch. Epoch is how many times the model looks at the data.</p><p>Machine learning has some limitations:</p><ol style="list-style-type: lower-alpha;"><li>A model can not be created without data</li><li>It can learn the patterns from input data</li></ol><hr /><h2>Deep Learning Vocabulary</h2><p>Label - a data we are trying to predict</p><p>Architecture - a template of a model we are tring to fit a model; an actual function</p><p>Parameters - values in model that change what task it can do and updates throughout the training process.</p><p>Fit - update the parameters such that predictions of model using input to match output labels. Also called Train.</p><p>Fine Tune - special technique in transfer learning to update pre-trained model.</p><p>Epoch - one complete task to pass through input data</p><p>Loss - a measure how good model is using validation set</p><p>Training set - data for training</p><p>Validation set - data for measuring loss</p><p>Overfitting - training remembers specific features of input than generalizing pattern.</p><p>CNN - neural network for computer vision</p>
